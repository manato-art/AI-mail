# 統合優先度一覧 — 営業メール自動送信ツール 追加機能候補155件

**総括**
既存32機能は「作る・送る・止める」の主要動線を押さえているが、実運用に落とすと**（1）送信事故・法令違反を即座に防ぐ薄いガード群、（2）2〜3名+エンジニア1名という小体制が壊れた時に詰む運用基盤（バックアップ・引き継ぎ・失敗リカバリ）、（3）成果を測るための最小限のパイプライン管理**が丸ごと抜けている。155件の大半は良質だが、その半分近く(48件)は「データが貯まる前」「人数が増える前」の段階では明確に費用対効果が合わない先取り投資であり、正直にCへ落とした。

## 最初の3ヶ月で作るべきもの

**Sの1〜16全て**（プロンプト版管理#16のみ4ヶ月目にずれ込んでも可）を最優先で通す。理由は、これらが「実装が軽い（大半が軽〜中）」にもかかわらず「事故・法令違反・データ消失・人員離脱」という一発アウト級のリスクを塞ぐか、2〜3名で日次200通を回し続けるための物理的前提だから。特に**出力スキーマ検証ゲート・誤送信防止セーフガード・リスト事前衛生チェック・DBバックアップ・失敗ログ再実行ビュー**の5つは実装コストが最も低くリスク低減効果が最も大きく、初月内での着手を推奨する。

Sが埋まった後、4ヶ月目以降にA①（レビュー効率化）とA②（到達性拡張）から着手し、A⑤（経営指標）とA④（AI品質）はSの案件パイプライン管理・プロンプト版管理が数週間データを溜めてから本格稼働させる。
