## (a) 公式API・構造化データ経由

### Indeed
- **現状**: Indeedは求人検索用の「Publisher API（求人検索API）」を2023年に廃止しており、現在は自己登録（セルフサーブ）で求人データを取得できるAPIキーは存在しない。さらに2025年6月30日をもって、企業サイト等の自動クローリング掲載も終了した。
- **現在提供されているAPI群**（Job Sync API、Disposition Sync API、Send Candidates API等）は全て「企業側がIndeedに求人・応募情報を送り込む」ための求人主・ATSベンダー向けAPIであり、「求人一覧を検索・取得するAPI」は含まれない。パートナープログラムへの審査通過が前提で、営業リスト用途の第三者には門戸が開いていない。
- **取得できる項目/コスト/更新頻度**: 該当なし（一般提供の求人検索APIが存在しないため）。
- **法的安全度**: 該当APIがないため評価不可。スクレイピングで代替する場合はIndeed利用規約違反リスクが高い。
- **推奨度**: ★1（事実上の入手不能ルート）

Sources: [Indeedのクローリング掲載が終了（2025年6月末）](https://bizpla.com/blog/indeedcrawlingend/) / [Indeed API: how to pull job postings from Indeed without scraping](https://jobspipe.dev/blog/indeed-api-guide) / [Job Sync API ガイド | Indeed Partner Docs](https://docs.indeed.com/job-sync-api/job-sync-api-guide)

### 求人ボックス（カカクコム運営）の「求人検索API（パブリッシャープログラム）」
- **現状**: 2023年7月開始。外部サイト運営者が自社サイト上に求人ボックスの求人（500万件以上）をJSON形式で表示するための無料API。契約前に掲載サイトの審査あり。
- **重要な制約（利用規約）**: 同社の利用規約には、提供情報を事前同意なく複写・複製・転売・第三者提供する行為の禁止、および「営業活動その他の営利を目的とした利用又はそれに準ずる行為」の禁止が明記されている（当方によるWebFetch要約に基づく。正確な条数は公式ページで再確認推奨）。**これは「自社サイトで求人を表示してアフィリエイト収益を得る」ための媒体パートナー向けAPIであり、社内の営業リストDB構築・コールドメール送信リストの元データにする用途は規約上想定外・抵触の可能性が高い**。
- **取得できる項目**: 企業名等は含まれるとみられるが、企業URL・応募先メールアドレスの有無はページ上に明記なし（未確認）。
- **コスト**: 無料（クリック報酬型・審査あり）。
- **更新頻度**: 未確認。
- **法的安全度**: 低〜中（規約上、営業目的・第三者提供が明示的に禁止されているため、そのまま営業リストに転用すると契約違反のリスク）。
- **推奨度**: ★1（用途がミスマッチ）

Sources: [求人検索API｜求人ボックス](https://xn--pckua2a7gp15o89zb.com/求人検索API) / [プレスリリース](https://prtimes.jp/main/html/rd/p/000000920.000001455.html) / [利用規約](https://xn--pckua2a7gp15o89zb.com/利用規約)

### スタンバイ（Stanby、LINEヤフー×ビズリーチ合弁）
- **現状**: 求人データの取得（インバウンド）方式としてクローリング・XMLフィード・自社投稿の3種があるが、これらは**企業・ATSがスタンバイに求人を「送り込む」ための仕組み**であり、外部の第三者がスタンバイの求人データを取得するための公開APIは確認できなかった。
- **取得できる項目/コスト/更新頻度**: 該当API不在のため評価不可。
- **法的安全度**: 該当なし。
- **推奨度**: ★1（第三者向け取得APIなし＝未確認だが現状は確認できず）

Sources: [スタンバイの検索の仕組み - Stanby Tech Blog](https://techblog.stanby.co.jp/entry/stanby_search) / [スタンバイに無料掲載する方法は3つ](https://saiyo-kakaricho.com/wp/how-to-post-the-ads-on-standby/)

### Google しごと検索（Google for Jobs）／Cloud Talent Solution
- **位置づけ**: 「しごと検索」は各社が自サイトのJobPosting求人ページに埋め込んだschema.org構造化データをGoogleがクロールして検索結果に表示する仕組みであり、**Googleが第三者向けに求人データを配信するAPIは存在しない**。JobPosting構造化データは「1ページに1求人」がGoogleのガイドライン要件であり、そもそも大量一括取得を想定した設計ではない。
- **Cloud Talent Solution**（Google Cloud製品）は求人主・ATS・求人サイト事業者が自社の求人データをGoogleのAI検索エンジンに載せて自社サイト内検索UIを作るための有償API群であり、**他社の求人情報を検索・取得する用途のAPIではない**（データは利用者自身がアップロードしたものに限定、テナント間分離あり）。
- **取得できる項目/コスト/更新頻度**: 該当なし。
- **法的安全度**: 該当なし。
- **推奨度**: ★1（求人ボックス同様、他社求人情報の取得手段としては機能しない）

Sources: [Learn About Job Posting Schema Markup | Google Search Central](https://developers.google.com/search/docs/appearance/structured-data/job-posting) / [Cloud Talent Solution | Google Cloud](https://cloud.google.com/solutions/talent-solution?hl=ja)

## (b) ハローワーク・厚労省の公的データ

### ハローワーク求人・求職情報提供サービス（API/CSV一括提供）
- **利用対象者（重要な制約）**: 「地方自治体・公共職業能力開発施設又は職業紹介事業者（厚生労働大臣の許可・届出番号がある方）」に限定。**一般の営業リスト会社・事業会社は、職業紹介事業の許可または届出（無料職業紹介事業の届出でも可）を取得しない限り、このサービスを直接利用できない**。
- **申込方法**: オンライン仮登録後、管轄労働局への利用申請書提出が必要。
- **データ形式**: 求人者（企業）が同意した有効求人をXML形式（1000件単位、都道府県別ファイル）で提供。更新は「前日ハローワーク受理分を毎日午前6時頃／21時30分時点で更新」との情報あり（情報源により時刻の記載が異なる、要再確認）。
- **項目・制限**: 具体的な収録項目（企業名・URL等）はAPIインタフェース仕様書（PDF）に記載されているが、本調査では詳細抽出未完了（未確認）。**利用規約上、職業紹介と無関係な目的での利用禁止、求人団体以外への無断提供禁止、内容の改変禁止**が定められている。新卒/中途/パート/インターンの区分の有無も未確認。
- **コスト**: 記載箇所を確認できず（未確認、無料の可能性が高いが要確認）。
- **法的安全度**: 高（許可取得後は完全に適法な一次データ）だが、**入口として職業紹介事業許可のハードルがある**ため一般営業リスト用途には現実的でない。
- **推奨度**: ★2（職業紹介事業の許可・届出を別目的でも取得する予定があるなら★4に上がるが、それ自体が大きなハードル）

## (d) プレスリリース・site:検索等の手動発見

- **実務**: `site:prtimes.jp インターン募集`、`site:hrpro.co.jp`、`"インターン募集" site:.co.jp`等のGoogle検索演算子、Xやウォンテッドリー等での「#採用強化中」投稿、マイナビ2028/ONE CAREER等の学生向けインターン掲載サイトを人力で巡回する方法。
- **法的安全度**: 高い。検索エンジンの通常利用や、企業が自ら公開しているプレスリリース・採用ページの閲覧は著作権法上も問題なく、公開情報の**人力**での収集は最も安全。ただし「大量に自動巡回してリスト化する」場合は各サイトのrobots.txt・利用規約の確認と、著作権法47条の5（情報解析目的の複製）の要件充足が必要になり、要件充足は「意外にハードルが高い」との弁護士解説がある。
- **コスト**: 実質無料だが人件費（時間）がかかる。1日に発見できる件数は数件〜数十件程度が現実的で、50〜200件/日の主力にはなり得ない。
- **推奨度**: ★3（量産には不向きだが、精度の高いターゲット選定・一次情報確認のダブルチェック用途として有用）

## (e) その他の現実的手段

1. **合同企業説明会・逆求人イベントの参加企業リスト**（マイナビ／ONE CAREER／Wantedly主催のインターンフェア等）は主催者が公式サイトで参加企業一覧を公開しているケースが多く、site:検索と組み合わせて手動収集可能。合法性は高い。
2. **大学キャリアセンターの求人票検索システム**（一部大学はログイン不要で企業名を確認できる場合あり）。ただし多くは学内限定でアクセス制限あり、対象外となるケースが多い。
3. **経済団体・商工会議所・自治体の「新卒応援ハローワーク」「わかものハローワーク」求人検索**（一般公開ページ、人力閲覧は自由）。
4. **有料職業紹介事業の許可・届出を自社で取得する**という中長期の選択肢。取得すればハローワーク求人情報オンライン提供サービス（(b)参照）が利用可能になり、一次情報を合法的にAPI/CSVで得られる。ただし許可要件・時間コストが発生するため、営業リスト調達だけを目的に取るのは費用対効果が悪い可能性が高い（要個別検討、未確認事項含む）。

## 法的な補足（横断的に効いてくる論点）

- **特定電子メール法（オプトイン規制の例外）**: BtoBで「Web上に自己のメールアドレスを公表している事業者」宛は、事前同意なしの送信が認められる例外規定がある。ただし当該メールアドレスの公表箇所に「営業メール送信を求めない」旨の記載がある場合は例外の対象外になる、という要件がある。件名・本文中の表示義務（送信者情報、配信停止方法の表示等）は例外の場合でも別途遵守が必要。
  Source: [特定電子メールの送信等に関するガイドライン｜総務省](https://www.soumu.go.jp/main_sosiki/joho_tsusin/d_syohi/pdf/m_mail_081114_1.pdf)
- **個人情報保護法**: 名刺交換等で取得した担当者個人のメールアドレスは「取得の状況からみて利用目的が明らかである場合」に該当すれば、別途の通知・公表なしに営業メール送信の利用目的として扱える、というのが個人情報保護委員会の公式見解。ただし**第三者（他社）へリストとして提供する場合は本人同意またはオプトアウト手続きが別途必要**。企業の代表アドレス（info@等、特定個人と紐付かないもの）は一般に「個人情報」に該当しない可能性が高いが、担当者名付きのメールは個人情報になり得る点に注意。
  Source: [個人情報保護委員会 FAQ（名刺交換により取得した連絡先への広告宣伝メール）](https://www.ppc.go.jp/all_faq_index/faq1-q4-16/)
- **著作権法47条の5**: スクレイピング自体を包括的に適法化する条文ではなく、要件（情報解析目的、軽微利用等）を個別に充足する必要があり、実務上ハードルが高いとの弁護士解説が複数ある。求人媒体各社の利用規約（前述の求人ボックスの例）でも複製・転売・営利目的利用を明示的に禁止しているケースがあり、**媒体ToS違反は著作権法とは別に契約法上のリスクとして独立に存在する**点に注意。
  Source: [スクレイピングはどこまで許されるのか | IT弁護士 大阪](https://www.ys-law.jp/IT/column/column-11384/)

## 推奨パイプライン構成（1日50〜200通のコールドメール供給を支える構成案）

**結論**: 「求人媒体の直接スクレイピング／利用規約違反リスクのあるツール（Urizo等）」への依存は避け、**商用企業DB（Musubu／BIZMAPS／SalesNowのいずれか、または併用）の公式「採用中」絞込み機能を主軸**にし、**手動site:検索・プレスリリース発見を精度担保のサブルートとして併用**する構成を推奨する。

1. **主軸（供給量の大部分・80〜90%）**: Musubu または BIZMAPS（もしくは両方を並行契約し重複除去）で「直近◯ヶ月以内に求人媒体へ出稿」「新卒/中途採用実施中」タグで軽井沢近隣・ターゲット業種の企業を毎週抽出 → 企業名・URL・代表連絡先をCSVでエクスポート。SalesNowのLite（従量課金）を併用し、条件を変えて追加分を都度購入するのも有効（契約縛りなしのため小規模運用に向く）。
2. **精度担保・穴埋め（供給量の10〜20%）**: (d)の手動site:検索・プレスリリース巡回・合同説明会参加企業リストで、DB化されていない/更新が遅い企業（特に地場の新規開業店・地方企業）を週1〜2時間の人力作業で補完。
3. **連絡先の扱い**: 企業DBから得られるメールアドレスが「担当者個人名付き」か「代表窓口（info@等）」かを仕分けし、個人名付きの場合は個人情報保護法上の取扱い（利用目的の妥当性）を意識する。送信文面には特定電子メール法が求める表示事項（送信者名・配信停止方法等）を必ず含める。
4. **職業紹介事業許可の取得は現時点では不要**と判断する（(b)のハローワーク公式APIはリスト調達だけを目的に取得するには許可取得コストが見合わない）。ただし将来的にインターン紹介・人材紹介事業へ本格展開するなら、許可取得を別目的の投資として再検討する価値はある。
5. **NGとする手段**: 求人ボックスAPI・スタンバイ由来データの営業目的転用（規約上禁止が明記）、Indeedの求人検索スクレイピング（API廃止済・規約違反リスク）、Urizo等の求人媒体クロール型ツールへの主軸依存（媒体側ToSとの緊張関係が構造的にある）。

**💬 ざっくり**: 「求人サイトから自動で企業リストを吸い上げるAPI」は実質もう存在しない（IndeedもGoogleも求人ボックスも、営業リスト作りに使うのは規約違反かそもそも不可能）ので、**お金を払って企業データベースサービス（Musubu・BIZMAPSなど）の「採用中企業」検索機能を使うのが一番安全で現実的**、という結論です。それだけでは拾いきれない地元の小さいお店などは、人がGoogle検索やプレスリリースを見て手作業で足すのが安全な補完策になります。ハローワークの公式データは魅力的に見えますが、「職業紹介の免許を持っている会社」しか使えない制度なので、今回の目的には向きません。
