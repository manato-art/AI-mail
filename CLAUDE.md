# 営業メール自動送信ツール — 開発ルール

> このファイルはリポジトリのルートに `CLAUDE.md` として置く。Claude Code が毎セッション自動で読み込む。

## このプロジェクトは何か

Next.js（App Router）+ SQLite（better-sqlite3）+ Claude API の営業メール作成ツール。
会社URLを貼ると、HPをクロール → AIが企業分析 → その会社向けの営業メールを生成する。

**要件書は `docs/spec-v2.md`。判断に迷ったらこれを正とする。** 調査の根拠は `docs/research/` 配下。

## 絶対に守る制約

1. **`send_log` は不変**。送信後に削除・更新しない。監査証跡であり二重送信判定の正
2. **送信APIを叩く前に「送信中」状態を永続化する**。逆順にするとクラッシュ再実行で二重送信が起きる
3. **`suppressions` チェックは無効化できない固定仕様**。配信停止対応は特定電子メール法上の法的義務
4. **未解決の `{{変数}}` が残っていたら送信をブロック**。目視前提にしない（KB: `prompt-rebuild-field-dropout`・再発2回）
5. **エラーを握り潰さない**。送信失敗・API エラー・検証不合格は必ず画面に理由を出す
6. **OAuthトークンは暗号化保存**（鍵は環境変数）。トークン・APIキーを画面/ログに出さない。外部APIのエラー詳細をUIに露出しない
7. **相手企業のHPは信頼できない外部入力**。HP内の指示文にAIが従わないよう、分析プロンプトはデリミタで囲い、生成段には分析JSONのみ渡す（生HTMLを渡さない）
8. **URL入力は http/https のみ許可**。プライベートIP・localhost へのリクエストを禁止（SSRF対策。既存 `lib/ssrf.ts`）

## コーディング規約

- **Immutability 優先**: 既存オブジェクトを mutate せず、新しいオブジェクトを返す
- **サイズ**: 関数 <50行 / ファイル 200〜400行目安（上限800行）/ ネスト ≤4（超えたら早期return・分割）
- **命名**: camelCase（変数・関数）/ PascalCase（型・コンポーネント）/ UPPER_SNAKE_CASE（定数）/ boolean は `is` `has` `should` `can` 接頭 / カスタムフックは `use` 接頭
- マジックナンバー・ハードコード値は名前付き定数か設定に出す
- **境界での入力検証必須**（外部データ・APIレスポンス・ユーザー入力を信用しない）
- production コードに `console.log` を残さない
- 既存のコードスタイル・ファイル構成に合わせる。新しい流儀を持ち込まない

## 主要ファイル

```
app/api/generate/route.ts       生成のコアAPI（クロール→分析→生成→品質チェック→保存）
app/prospect/[id]/page.tsx      確認画面。ここに「送信」を実装する
lib/db.ts                       SQLite。テーブル定義・マイグレーション・クエリ
lib/crawl.ts                    1社のHPクロール（メアド・フォームURL抽出も担当）
lib/analyze.ts                  企業分析（構造化JSON出力）
lib/generate.ts                 メール生成
lib/quality-check.ts            生成後の機械検証 ← 変数漏れチェックはここを拡張
lib/ssrf.ts                     SSRF対策
lib/types.ts                    型定義
```

## 作業の進め方

- 実装前に**変更ファイルと追加テーブルの一覧を提示**する
- 既存機能（生成・分析・テンプレ・履歴）を壊さない
- 要件書と矛盾する点・実装上おかしい点は、**勝手に解釈せず質問する**
- 大きめの変更は分割する。一度に全部やらない
- `git push` は勝手に実行しない。commit までに留め、push は人間が判断する
